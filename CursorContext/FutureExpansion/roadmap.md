# Suppli — Product Roadmap

## Purpose of This Document
This document outlines Suppli's product roadmap, feature priorities, and evolution beyond MVP.
The roadmap is a living document that guides development priorities while remaining flexible to user feedback and market needs.

This roadmap focuses on **value delivery, risk mitigation, and sustainable growth**.

---

## Roadmap Philosophy
1. **MVP first** — Core value before nice-to-haves
2. **User feedback drives priorities**
3. **Quality over speed**
4. **Measurable outcomes**
5. **Flexible timeline**

---

## MVP (Phase 0) — Foundation

### Goal
Launch a working product that delivers core value safely.

### Core Features
- ✅ Business & user onboarding
- ✅ Vendor management
- ✅ Sales data import (CSV/manual)
- ✅ Order generation (conservative, guided)
- ✅ Order review & approval
- ✅ Vendor formatting (email, printable, CSV)
- ✅ Invoice upload & basic verification
- ✅ Audit logging
- ✅ Basic billing (Stripe)

### Success Metrics
- Users complete onboarding
- Users generate and approve first order
- Edit rate decreases over time
- No catastrophic ordering mistakes

### Timeline
- Target: Launch-ready MVP

## Phase 1 — Trust & Accuracy (Months 1-3)

### Goal
Build user trust through improved accuracy and reliability.

### Features

#### Enhanced Learning
- Product-level learning adjustments
- Confidence calibration based on edits
- Vendor reliability tracking
- Promo performance history

#### Improved Data Ingestion
- POS integration (Square or Clover)
- Automatic sales data sync
- Better column mapping
- Data quality indicators

#### Invoice Verification Improvements
- Better OCR accuracy
- Price verification
- Mismatch resolution workflow
- Vendor follow-up suggestions

#### Order Generation Refinements
- Slightly less conservative caps (with confidence)
- Better promo uplift estimation
- Seasonal awareness (basic)

### Success Metrics
- Edit rate decreases 30%+
- Invoice mismatch detection accuracy >80%
- User retention improves
- Confidence levels increase

## Feature Prioritization Framework

### Priority Factors
1. **User value** — How much does this help users?
2. **Risk reduction** — Does this prevent costly mistakes?
3. **Retention impact** — Will this keep users?
4. **Differentiation** — Does this set us apart?
5. **Feasibility** — Can we build this well?

### Priority Levels
- **P0** — Critical for MVP
- **P1** — High value, post-MVP
- **P2** — Medium value, nice to have
- **P3** — Future consideration

---

## Out of Scope (Explicitly)

### Not Planned
- Full inventory management
- Accounting system replacement
- Employee scheduling
- Customer-facing features
- Payment processing for vendors
- Marketplace features

### Why
- Focus on core value
- Avoid scope creep
- Maintain simplicity
- Partner with specialists

---

## Risk Mitigation

### Technical Risks
- **Data quality** — Mitigate with validation and user feedback
- **Integration reliability** — Fallback to manual uploads
- **Scale issues** — Design for scale from day one

### Product Risks
- **Low accuracy** — Conservative approach, guided mode
- **Low adoption** — Focus on onboarding and value delivery
- **Feature bloat** — Strict prioritization, user feedback

### Business Risks
- **Competition** — Focus on trust and accuracy
- **Market fit** — Validate early, iterate quickly
- **Churn** — Prevent bad experiences, support users
